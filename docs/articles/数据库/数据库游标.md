数据库游标（Cursor）是数据库系统提供的一种机制，用于逐行处理查询结果集，而不是一次性将所有数据加载到内存中。它相当于一个指向结果集中某一行的“指针”，开发者可以通过游标逐行读取、更新或删除数据。

### 游标的核心概念
1. **声明（DECLARE）**：定义游标并绑定一条 SELECT 语句。  
2. **打开（OPEN）**：执行绑定语句，把结果集放入内存工作区。  
3. **提取（FETCH）**：将游标指针下移一行，并把该行数据读入变量。  
4. **关闭（CLOSE）**：释放游标占用的资源。  
5. **释放（DEALLOCATE）**：彻底销毁游标对象（部分数据库需要）。

### 使用场景
- 需要逐行处理大量数据，避免一次性加载导致内存溢出。  
- 在存储过程或触发器中对每一行做复杂业务判断。  
- 需要可滚动（前后移动）或敏感（感知并发更新）的访问模式。
